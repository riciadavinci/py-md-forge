I am planning to make a Python CLI tool to compile Markdown files, specifically the README.md file for my projects on Github. I would not be creating an explicit README.md file for projects, instead, I will create a docs directory with a TEMPLATE.md file that will contain special syntax to include sub-files (which are markdown files themselves). An example of TEMPLATE.md would be something as follows:

```plaintext - TEMPLATE.md

{{DESCRIPTION}}

{{TABLE_OF_CONTENTS}}

{{SECTION_1}}

{{SECTION_2}}

{{SECTION_3}}

.
.
.

{{SUMMARY}}

```
This input Template file would be passed as a required CLI argument. All the sub-files should follow the names given within "{{}}". If files are not found, it will throw an error.

I will also have special syntax for images within these sub-files. Example:
```plaintext - DESCRIPTION.md

MARKDOWN_IMAGE_LINK=/filename.png

```
This will be replaced with the input image path passed as another required CLI argument.

The project root would look something like this:
/sample-project

  ├──src/ 			# It might contain source files
  ├──deps/ 			# It might contain external dependencies

  │  ├──TEMPLATE.md
  │  ├──TOC.md
  │  ├──SECTION_1.md
  │  ├──SECTION_2.md
  │  └──CONCLUSION.md
  └──@py-md-compiler/ # The project we are discussing included as a sub-module (can also be placed under utils/ dir)
